# Procore Marketplace Opportunity Analysis
**Saved: March 16, 2026**

## Concept: RMS → Procore Migration Tool

### The Competition

**The Link Submittal Creator** (https://marketplace.procore.com/apps/the-link-submittal-creator):
- Creates submittal logs from specifications (standard commercial AND UFGS/military specs)
- Founded by "former construction specifications and general contractor leaders"
- Live on marketplace since April 2024
- Requires separate license from The Link (pricing not public)
- Does NOT appear to be a full RMS migration tool - just spec parsing

**Key insight:** Nobody appears to be doing a complete RMS-to-Procore migration. That's the niche.

### Target Market

USACE contractors who:
- Are required to use RMS (https://rms.usace.army.mil) for government compliance
- Also use Procore for their internal project management
- Currently duplicate data entry between both systems

Market size indicators:
- USACE awards billions annually across hundreds of contracts
- Contractors range from small firms to giants like Clark Construction, AECOM, etc.
- Many use both systems in parallel (pain point being solved)

### Procore Partner Program

From Procore's partner info (https://developers.procore.com/partner):
- 539+ apps currently on marketplace
- Tiered partner program with marketing resources, developer support
- Apps must pass "strict technical requirements"
- Fees/revenue share: Not publicly disclosed - need to contact them directly

### Proposed App Features

**"RMS to Procore Migration Suite"** could include:
1. Submittal Register import (BUILT)
2. Submittal Assignments import
3. Bulk file upload to submittals (BUILT)
4. Custom field population
5. Directory/Company matching (subcontractor sync)
6. Potentially: QC reports, daily logs, other RMS data

### Assessment

| Factor | Assessment |
|--------|------------|
| Is there a market? | Yes - every USACE contractor using Procore has this pain |
| Is there competition? | Minimal - The Link does spec parsing, not RMS migration |
| Is it technically feasible? | Yes - core functionality already proven |
| Support burden? | Medium - RMS exports are standardized, but edge cases exist |
| Revenue model? | Per-migration fee? Monthly subscription? Per-project? |
